`critique` matters because a page can be visually refined and still be conceptually weak. If the hierarchy is confusing, the task sequence is backwards, or the layout reads like generic filler, polish will only make the mistake look cleaner.

The workflow uses critique early so the important problems get fixed before the final pass.
